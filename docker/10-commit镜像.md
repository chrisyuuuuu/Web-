### run起来会产生两层
	- 容器层（所有操作）
	- 镜像层（只读）
### commit
	- 容器层+镜像层打包提交
	```
		docker commit 提交容器成为一个新的副本
		docker commit -m="提交的描述信息" -a="作者" 容器id 自定义目标镜像名:[tag]
	
	### 实战测试
		1. 启动一个默认的tomcat 
		2. 是没有webapps应用，镜像的原因，webapps下是没有文件的
		docker ps
		3. 拷贝进去了基本的文件
		4. 将我们操作过的容器通过commit提交为一个镜像
		
	```
	- 注意想保存当前容器的状态，就可以通过commit来提交，获得一个镜像，好比VM时候的快照
